# DatabasesISTU — аэропорты и маршруты

Курсовая работа по Базам Данных 2 курс Мошкин Владимир Б24-191-3
Приложение поддерживает универсальную работу с таблицами базы данных. Для этого используются запросы к системным представлениям information_schema и служебным каталогам PostgreSQL, позволяющие получить перечень таблиц, сведения о столбцах, типах данных, признаке обязательности, значениях по умолчанию и составе первичных ключей. Такой подход делает приложение менее зависимым от жестко зафиксированной структуры базы данных и позволяет применять единый механизм обработки для разных таблиц.
Разработанное приложение обеспечивает выполнение основных операций обработки данных. Пользователь может просматривать содержимое таблиц, добавлять новые записи, изменять существующие и удалять их. Для таблицы маршрутов реализован специальный режим ввода, при котором одновременно добавляется основная запись о маршруте и набор связанных транзитных стоянок. Такая операция выполняется в рамках одной транзакции, что гарантирует согласованность данных.
Помимо операций редактирования, приложение предоставляет возможность формирования аналитических отчетов. В частности, реализованы запросы для получения статистики по количеству мест размещения в разных населенных пунктах, средней стоимости проживания по типам мест размещения и перечня маршрутов с указанием числа транзитных стоянок. Наличие таких функций делает клиентское приложение не только средством ввода данных, но и инструментом анализа информации, хранящейся в базе данных.
Интерфейс приложения имеет консольный формат и организован в виде системы меню. Пользователю предоставляются понятные варианты выбора действий, сообщения о результатах операций и уведомления об ошибках. При вводе данных выполняется базовая проверка значений, а при завершении работы предусмотрена корректная обработка прерывания ввода. Это делает использование приложения удобным и понятным даже при отсутствии графической оболочки.

## Состав проекта

### Файлы

| Файл                   | Назначение                                                                     |
| ---------------------- | ------------------------------------------------------------------------------ |
| `schema.sql`           | Создание и обновление схемы БД: таблицы, индексы, VIEW, функции и триггеры     |
| `seed.sql`             | Очистка и заполнение тестовыми данными                                         |
| `db.py`                | Подключение к PostgreSQL через `psycopg`, вспомогательные запросы к метаданным |
| `app.py`               | Консольный клиент: CRUD по таблицам, форма 1:M, отчёты                         |
| `postgres.session.sql` | Заготовка под сессию в IDE (по желанию)                                        |

### Таблицы

| Таблица          | Описание                                                                            |
| ---------------- | ----------------------------------------------------------------------------------- |
| `airports`       | Аэропорты (ключ: `airport_code`; дополнительно счётчики вылетов/прилётов)           |
| `airplanes`      | Самолёты, привязанные к аэропорту (составной ключ: `airport_code`, `airplane_code`) |
| `positions`      | Должности                                                                           |
| `crews`          | Экипажи                                                                             |
| `services`       | Службы                                                                              |
| `personnel`      | Персонал (составной ключ: `airport_code`, `person_inn`)                             |
| `routes`         | Маршруты/рейсы (ключ: `route_code`; счётчик транзитных остановок)                   |
| `transit_routes` | Транзитные остановки по маршруту                                                    |

### Представления (VIEW)

| Представление               | Смысл                                                    |
| --------------------------- | -------------------------------------------------------- |
| `v_airports_catalog`        | Справочник аэропортов (одна таблица + счётчики)          |
| `v_route_schedule`          | Рейсы с названиями аэропортов и борта (несколько таблиц) |
| `v_airports_departures_agg` | Агрегаты по вылетам с `GROUP BY` и `HAVING`              |

### Триггеры и денормализация

- **`tr_transit_routes_denorm`** — при изменении `transit_routes` обновляет `routes.transit_stop_count`.
- **`tr_routes_denorm`** — при изменении маршрутов обновляет счётчики в `airports` и `airplanes` (при смене аэропортов вылета/прилёта или назначенного самолёта; при правке только «операционных» полей лишних пересчётов по аэропортам нет).

### Возможности приложения (`app.py`)

- **Общее меню**: все перечисленные таблицы + отчёты + форма ввода в две таблицы с отношением **1:M** (маршрут `routes` и строки `transit_routes`).
- **Режим таблицы**: перемещение по записям и страницам, добавление, редактирование, удаление (где есть первичный ключ), поиск по полю, фильтр, сортировка.
- **Отчёты (не менее трёх)**: маршруты по конечному пункту, ближайшие рейсы, персонал по службам/должностям; перед построением запрашиваются фильтры и параметры сортировки; в отчётах используются соединения таблиц, группировка, вычисляемые поля и итоги.
  Команды в режиме таблицы:
- `n` / `p` — следующая / предыдущая запись в текущей выборке
- `next` / `prev` — следующая / предыдущая страница
- `a` — добавить, `e` — редактировать, `d` — удалить
- `s` — поиск (ILIKE), `f` — фильтр, `o` — сортировка, `r` — обновить выборку, `q` — назад в меню
