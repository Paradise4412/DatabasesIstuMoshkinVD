## Инструкция по запуску

### Что должно быть установлено

- **PostgreSQL** (сервер запущен, доступна утилита `psql`).
- **Python 3** (рекомендуется отдельное виртуальное окружение из-за ограничения PEP 668 в системных установках Python на macOS через Homebrew).
  В терминале перейдите в каталог проекта и создайте базу (имя можно заменить на своё; ниже используется `airdb`):

```bash
cd /path/to/DatabasesISTU
createdb airdb
psql -d airdb -f schema.sql
```

Скрипт `schema.sql` создаёт таблицы, ограничения, индексы, представления (VIEW) и триггеры для денормализованных счётчиков. Повторный запуск обычно безопасен: недостающие столбцы и объекты добавляются через `IF NOT EXISTS` / `CREATE OR REPLACE` там, где это предусмотрено.
Чтобы в таблицах сразу были строки и отчёты показывали результат:

```bash
psql -d airdb -f seed.sql
```

Скрипт очищает данные в таблицах и заново заполняет аэропорты, справочники, самолёты, персонал, маршруты и одну транзитную стоянку. Время вылета части рейсов задано относительно `now()`, чтобы отчёт «ближайшие рейсы» находил строки при окне в несколько часов.
Установите пакеты **в виртуальное окружение** (так вы обойдёте ошибку `externally-managed-environment` при глобальном `pip`):

```bash
cd /path/to/DatabasesISTU
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

Если каталог `.venv` уже создан ранее, достаточно активировать его и выполнить `pip install -r requirements.txt`
риложение читает переменную **`DATABASE_URL`** (при необходимости можно использовать **`PG_DSN`** с тем же форматом).
Пример для локального пользователя без пароля:

```bash
export DATABASE_URL="postgresql://$USER@localhost:5432/airdb"
```

Если у роли задан пароль:

```bash
export DATABASE_URL="postgresql://USER:PASSWORD@localhost:5432/airdb"
```

Подставьте своего пользователя, хост, порт и имя базы.
С активированным venv и заданным `DATABASE_URL`:

```bash
python app.py
```

Дальше следуйте пунктам меню: работа с таблицами, форма ввода «маршрут + транзит» (1:M), отчёты.

## Состав проекта

### Файлы

| Файл                   | Назначение                                                                     |
| ---------------------- | ------------------------------------------------------------------------------ |
| `schema.sql`           | Создание и обновление схемы БД: таблицы, индексы, VIEW, функции и триггеры     |
| `seed.sql`             | Очистка и заполнение тестовыми данными                                         |
| `db.py`                | Подключение к PostgreSQL через `psycopg`, вспомогательные запросы к метаданным |
| `app.py`               | Консольный клиент: CRUD по таблицам, форма 1:M, отчёты                         |
| `requirements.txt`     | Зависимость: `psycopg[binary]`                                                 |
| `postgres.session.sql` | Заготовка под сессию в IDE (по желанию)                                        |

### Таблицы

| Таблица          | Описание                                                                            |
| ---------------- | ----------------------------------------------------------------------------------- |
| `airports`       | Аэропорты (ключ: `airport_code`; дополнительно счётчики вылетов/прилётов)           |
| `airplanes`      | Самолёты, привязанные к аэропорту (составной ключ: `airport_code`, `airplane_code`) |
| `positions`      | Должности                                                                           |
| `crews`          | Экипажи                                                                             |
| `services`       | Службы                                                                              |
| `personnel`      | Персонал (составной ключ: `airport_code`, `person_inn`)                             |
| `routes`         | Маршруты/рейсы (ключ: `route_code`; счётчик транзитных остановок)                   |
| `transit_routes` | Транзитные остановки по маршруту                                                    |

### Представления (VIEW)

| Представление               | Смысл                                                    |
| --------------------------- | -------------------------------------------------------- |
| `v_airports_catalog`        | Справочник аэропортов (одна таблица + счётчики)          |
| `v_route_schedule`          | Рейсы с названиями аэропортов и борта (несколько таблиц) |
| `v_airports_departures_agg` | Агрегаты по вылетам с `GROUP BY` и `HAVING`              |

### Триггеры и денормализация

- **`tr_transit_routes_denorm`** — при изменении `transit_routes` обновляет `routes.transit_stop_count`.
- **`tr_routes_denorm`** — при изменении маршрутов обновляет счётчики в `airports` и `airplanes` (при смене аэропортов вылета/прилёта или назначенного самолёта; при правке только «операционных» полей лишних пересчётов по аэропортам нет).

### Возможности приложения (`app.py`)

- **Общее меню**: все перечисленные таблицы + отчёты + форма ввода в две таблицы с отношением **1:M** (маршрут `routes` и строки `transit_routes`).
- **Режим таблицы**: перемещение по записям и страницам, добавление, редактирование, удаление (где есть первичный ключ), поиск по полю, фильтр, сортировка.
- **Отчёты (не менее трёх)**: маршруты по конечному пункту, ближайшие рейсы, персонал по службам/должностям; перед построением запрашиваются фильтры и параметры сортировки; в отчётах используются соединения таблиц, группировка, вычисляемые поля и итоги.
  Команды в режиме таблицы:
- `n` / `p` — следующая / предыдущая запись в текущей выборке
- `next` / `prev` — следующая / предыдущая страница
- `a` — добавить, `e` — редактировать, `d` — удалить
- `s` — поиск (ILIKE), `f` — фильтр, `o` — сортировка, `r` — обновить выборку, `q` — назад в меню
